# PRD-Flow-Interactions：游戏流程与主交互需求

---

## 1. 游戏主流程

1. 玩家进入标题页
2. 点击开始游戏
3. 进入主游戏场景
4. 玩家点击当前岸角色上船
5. 玩家点击开船
6. 船移动到对岸
7. 角色自动下船
8. 系统检查胜负
9. 未结束则继续下一步；胜利则进入胜利结算；失败则播放失败演出后进入失败结算
10. 玩家可选择重开、撤销或返回标题

---

## 2. 单步操作流程

一次完整移动流程：
1. 玩家选择角色上船
2. 系统检查船容量
3. 玩家点击开船
4. 系统检查船上是否有人
5. 锁定输入
6. 播放开船动画
7. 更新船的位置
8. 船上角色自动下船
9. 更新左右岸人数
10. 步数 +1
11. 写入历史记录
12. 检查失败
13. 检查胜利
14. 解锁输入或进入结算

---

## 3. 交互需求

### 3.1 点击岸上角色上船

条件：角色必须在船当前所在岸；船上人数 < 2；游戏状态为 playing；船没有正在移动；AI 演示没有运行。

行为：角色进入船上选择状态；角色移动到船上空座位；船上座位显示更新；开船按钮状态更新。

非法提示：船不在这一岸 / 船最多只能坐 2 个人 / 船正在动，先别急 / 现在不能操作。

### 3.2 点击船上角色取消上船

条件：角色已经在船上；船尚未开动；游戏状态为 playing。

行为：角色回到当前岸队列；释放船上座位；开船按钮状态更新。

### 3.3 点击开船

条件：船上至少 1 人；船上最多 2 人；游戏状态为 playing；船没有移动中；AI 演示没有运行。

行为：锁定输入 → 播放开船动画 → 船移动到对岸 → 角色自动下船 → 更新状态 → 记录历史 → 步数 +1 → 检查失败 → 检查胜利 → 进入下一状态。

非法提示：船上没人，开不了船 / 这船最多坐 2 个人 / 船正在动，先别急 / 现在不能开船。

失败处理：移动后触发失败则播放失败演出，然后进入失败结算。

### 3.4 撤销

行为：恢复到上一步完整状态；同步恢复船位置、角色位置、人数、步数、历史记录；清除提示高亮；失败状态也允许撤销回到失败前一步。

限制：没有历史状态时撤销按钮置灰；船移动过程中不能撤销；AI 演示过程中不能撤销，除非先停止演示。

提示文案："已经没有上一步了。"

### 3.5 提示

行为：调用 BFS 从当前状态获取最短路径 → 取第一步作为推荐 → 高亮推荐角色 → 显示提示文案。

提示文案示例："试试让 2 个华强上船。" / "试试让 1 个华强回来。" / "试试让 2 个郝哥上船。" / "试试让 1 个华强和 1 个郝哥回来。"

无解提示："当前局面无解，建议撤销一步。"

要求：提示必须基于当前状态实时计算，不允许写死；撤销后再次点击提示必须重新计算。

### 3.6 AI 自动演示

行为：点击"AI 演示"后进入自动模式；系统调用 BFS 获取最短安全路径；依次执行每一步：自动选择角色 → 播放上船动画 → 自动触发开船逻辑 → 船移动 → 角色自动下船 → 等待 0.8~1.2 秒 → 执行下一步。

限制：AI 演示期间禁用玩家点击；提供"停止演示"按钮；停止后保留当前状态，玩家可继续手动操作；AI 演示过程中不能撤销，除非先停止演示。
