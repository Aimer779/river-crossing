# PRD-UI：界面、历史记录与响应式适配

---

## 1. 顶部状态栏

显示内容：左岸：华强 × N，郝哥 × N；右岸：华强 × N，郝哥 × N；船上：N / 2；步数：N。

示例：左岸：华强 × 3  郝哥 × 2；右岸：华强 × 0  郝哥 × 1；船上：1 / 2；步数：4。

---

## 2. 操作按钮

按钮列表：开船、撤销、提示、AI 演示、停止演示、重新开始、静音、返回标题。

按钮状态：Normal / Hover / Pressed / Disabled。

禁用规则：
- 船为空时，开船按钮 disabled
- 没有历史时，撤销按钮 disabled
- AI 演示中，除"停止演示"和"静音"外，其他按钮 disabled
- 船移动中，大部分按钮 disabled
- 游戏结算后，普通操作按钮 disabled

---

## 3. 非法操作提示

不使用浏览器 alert，采用游戏内提示：点击位置浮动文字 / 底部 Toast / 角色旁气泡。

提示持续时间：0.8~1.2 秒；动画：轻微上浮 + 淡出，可伴随按钮抖动。

非法提示文案：船上没人，开不了船 / 这船最多坐 2 个人 / 船不在这一岸，别乱点 / 船正在动，先别急 / 现在不能操作 / 已经没有上一步了 / 当前局面无解，建议撤销一步。

失败提示文案：坏了，郝哥被拿捏了 / 这一岸华强太多，郝哥顶不住了 / 局势失控。

---

## 4. 历史记录

主界面显示最近若干步操作。

格式示例：第 1 步：2 个华强 → 右岸；第 2 步：1 个华强 ← 左岸。

要求：历史记录与状态栈一致；撤销后删除最后一步记录；AI 演示产生的步骤也要记录；超出显示区域后可滚动，或只显示最近 5 步。

---

## 5. 响应式适配

推荐逻辑画布：1280 × 720。

Phaser 配置建议：

```ts
scale: {
  mode: Phaser.Scale.FIT,
  autoCenter: Phaser.Scale.CENTER_BOTH
}
```

适配要求：
- 桌面浏览器正常显示；移动端建议横屏游玩；竖屏时显示提示"请横屏游玩"。
- 角色点击热区应大于视觉尺寸；移动端点击热区建议放大到 1.5 倍。
- UI 不要贴边，预留安全区域。
- 窗口 resize 后，船、角色、按钮位置不能错位。
