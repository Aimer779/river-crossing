# PRD-Test-Release：测试、验收与发布

---

## 1. 测试用例

### 1.1 基础流程

进入标题页 → 点击开始游戏 → 进入主场景 → 点击当前岸角色上船 → 点击船上角色取消 → 选择 2 个角色上船 → 点击开船 → 船到对岸 → 角色自动下船 → 步数 +1 → 历史记录增加。

### 1.2 非法操作

空船开船、船上超过 2 人、点击非当前岸角色、船移动中点击角色、AI 演示中点击角色、无历史时点击撤销、游戏结束后继续点击角色、快速连续点击同一角色、快速连续点击开船按钮。

### 1.3 失败流程

执行导致左岸不安全的移动；执行导致右岸不安全的移动；确认失败岸提示正确；确认失败动画播放；确认失败后进入失败结算；失败后点击撤销回到失败前一步。

### 1.4 胜利流程

按 BFS 路径完成游戏；确认进入胜利结算；确认步数正确；确认重新开始后状态清空；确认返回标题正常。

### 1.5 提示流程

初始状态点击提示；中间状态点击提示；撤销后点击提示；无解状态点击提示；确认提示始终来自当前 BFS 结果。

### 1.6 AI 演示

点击 AI 演示 → 自动完成完整路径；中途停止演示；停止后玩家可以继续操作；演示期间普通按钮禁用；演示结束后进入胜利结算。

## 2. 验收标准

### 2.1 玩法验收

玩家可正常进入游戏；可通过正确操作通关；可触发失败并看到失败原因；可撤销；可使用提示；可观看 AI 演示；所有操作都有清晰反馈。

### 2.2 状态验收

任意时刻满足人数守恒：
- `left.huaqiang + right.huaqiang + boat.huaqiang === 3`
- `left.haoge + right.haoge + boat.haoge === 3`

船上人数满足：`0 <= boat.huaqiang + boat.haoge <= 2`

船位置只能是：`'left' | 'right'`

游戏状态只能是：`'playing' | 'win' | 'lose'`

### 2.3 BFS 验收

初始状态必须有解；BFS 返回路径必须能执行到胜利；提示使用 BFS 第一步；AI 演示使用 BFS 完整路径；撤销后 BFS 必须基于新状态重新计算。

### 2.4 UI 验收

按钮状态准确；步数准确；历史记录准确；左右岸人数准确；船上座位准确；非法提示准确；失败岸提示准确；重开后所有状态清空。

## 3. 公开发布注意事项

内部 demo 可自由整活。公开发布建议注意：

- 不要直接使用真人肖像、影视截图、影视剧原声。
- 不要照搬大量经典台词。
- 角色形象应做原创化处理；语音建议自录或 TTS 生成原创短句。
- 整体表达保留梗感，但避免强绑定具体影视版权素材。

推荐做法：保留"华强 / 郝哥"的抽象梗感；美术使用原创 Q 版角色；台词使用原创变体；不要直接复刻具体影视画面。
